Analyzing the binary

Thanks to file command, we already know that the binary is not stripped which means that it contains symbols that will ease the analysis.

ghidra decompilation

Passing the binary to ghidra’s decompiler, we can retrieve code close to the original source code:

The binary use system function to run cat. But there is an issue. Indeed cat is called without its absolute path. So if we change our $PATH variable and name a malicious binary cat, it will be executed with mike’s privileges (since it’s a SUID binary).
